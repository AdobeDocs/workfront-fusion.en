# Workfront Fusion release activity: Week of June 1 2026

This page describes all enhancements made in Adobe Workfront Fusion the week of June 1 2026.

For a list of all recent changes, see [Adobe Workfront Fusion release activity](/help/workfront-fusion/fusion-product-releases/fusion-release-activity.md).

For a list of recent bug fixes in Workfront Fusion, see the [Workfront Maintenance Updates](https://experienceleague.adobe.com/en/docs/workfront-known-issues/releases/current-updates) page and check for any updates labeled Workfront Fusion Maintenance Update.


## Retrigger specific scenario executions

To allow you to apply scenario updates to previous scenario executions, we've made it possible to re-trigger webhook and child scenarios. Now, when you retrigger a scenario, the initial input for that scenario is pulled into a new execution. The new execution includes any updates to the scenario since the initial execution, such as added modules or other logic. 

For more information on retriggering scenarios, see [Retrigger a specific scenario execution](/help/workfront-fusion/manage-scenarios/retrigger-specific-execution.md).


## Updates to "Run once" functionality

To make it easier to test and troubleshoot webhook and child scenarios, we've added the ability to set input when using "Run once." Now, you can set input manually, use input from a previous scenario, or wait for an event that would trigger the scenario.

For more information, see [Use Run once to test a scenario](/help/workfront-fusion/manage-scenarios/run-once.md).
